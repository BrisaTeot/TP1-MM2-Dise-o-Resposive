Diseño Responsive

Un poco de historia

El concepto de diseño responsive fue planteado en 2010 por Ethan Marcotte, quien escribió un artículo que cambió la forma de pensar el diseño web. Antes de eso, era común hacer versiones diferentes de un mismo sitio: una para escritorio y otra para móviles. Su enfoque propuso usar los mismos principios del diseño gráfico flexible, pero llevados al entorno digital, aprovechando CSS3 y las capacidades de los navegadores modernos.

¿Qué es el diseño responsive?

El diseño responsive (o diseño adaptable) es una técnica de diseño web que permite que un sitio se adapte automáticamente a diferentes tamaños de pantalla y dispositivos, como celulares, tablets, laptops y monitores grandes. En lugar de crear versiones separadas para cada dispositivo, se desarrolla una sola página capaz de ajustarse según el entorno donde se visualiza.

¿Cómo se hace el diseño responsive?

Para lograrlo en lugar de hacer una sola versión fija de la web, usamos reglas condicionales en CSS llamadas media queries. Estas reglas permiten aplicar distintos estilos según el ancho del dispositivo u otras características como la orientación.

¿Qué son? Son reglas condicionales en CSS que permiten aplicar estilos distintos según el dispositivo.

¿Cómo se escriben?

@media (max-width: 768px) {
    body {
      background-color: lightblue;
    }
  }

¿Para qué sirven? Para que el contenido se vea bien tanto en celulares como en computadoras. Algunos ejemplos útiles:

  • max-width: 768px → para celulares
  • min-width: 769px → para pantallas grandes
  • orientation: portrait → si el dispositivo está en vertical

El diseño resposive y la transmedialidad

Una experiencia transmedia efectiva necesita que la historia o mensaje conserve su identidad, sin importar el medio. El diseño responsive ayuda a que los elementos visuales y de navegación mantengan consistencia, asegurando que cada dispositivo ofrezca una experiencia fluida, sin romper la inmersión del usuario en la narrativa.
